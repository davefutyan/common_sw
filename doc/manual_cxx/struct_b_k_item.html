Detailed Description

Defines one item for which the status is / can be booked.

Author
Reiner Rohlfs UGE

A BKItem is for example one visit.
The status of a BKItem can be either "exist" or "does not exist"
The list of kbNumbers has to be identical as defined for the bookkeeping group this item belongs to. The order of the BKNumbers as well as the BKNumber itself must fit (see also compare operator of BKNumber)
