Detailed Description

A class to access all parameters and sub-modules of a module.

Author
Reiner Rohlfs UGE

The only possible way to create an instance of this class is the method ProgramParams::Module(). Or if an instance of ModuleParams exists already the method ModuleParams::Module() can be used to create a ModuleParams.
This class is useful if the values of several parameters of a sub-module should be read. The module name of this module must not be specified in the full path of the parameter name. Following two code snippets are identical and illustrate the usage of this class: 

 ParamsPtr prog = CheopsInit(argc, argv);
 auto myIntVar = prog->GetAsInt("module1.param1");
 auto myBoolVar = prog->GetAsBool("module1.param2");

and 

 ParamsPtr prog = CheopsInit(argc, argv);
 const ModuleParams module = prog->module("module1");
 auto myIntVar = module->GetAsInt("param1");
 auto myBoolVar = module->GetAsBool("param2");

◆ GetAsBool()

bool ModuleParams::GetAsBool ( const std::string &  name) const
inline

Returns the value of a bool-parameter.

The value of parameters with data type == bool can be accessed with this function. Note: the data type is defined in the XML parameter file.
Following case insensitive strings are interpreted as true:
"true", "on", "yes", "1".
Following case insensitive strings are interpreted as false:
"false", "off", "no", "0".

Parameters
[in]namethe partial name of a parameter. If the parameter is defined in a sub-module of this module the sub-module name must be part of the name. For example: "sub-module.parameter". But the name of this module must be omitted.
Exceptions
runtime_errorif the full parameter name (including the module name of this module) does not exist or if the value of the parameter cannot be understood to be a bool variable.
exceptionif the parameter name is not of data type bool
Returns
value of the parameter.
