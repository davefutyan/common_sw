Detailed Description

Representing a FITS header with access functions to the attributes.

Author
Reiner Rohlfs UGE

Most of the times this FitsDalHeader is used as the parent class of the FitsDalImage and FitsDalTable classes to access a FTIS extension. But it is also possible to open any FITS extension, using this class to read the header keyword. In this approach it is not possible to update the header keywords or to access the data of the HDU.

FitsDalHeader::FitsDalHeader ( const std::string &  filename)

Constructor to open a FITS header for reading of the header keywords.

Opens a FITS data structure in read-only mode.

There is no access to the data of the HDU, i.e. to the image ore to the table. Only the header keywords can be read.

Parameters
[in]filenamefilename with optional HDU definition.
Exceptions
runtime_errorif the extension cannot be opened.

Writes all keywords in the FITS file, if in update mode and closes the file.

If the extension was created or opened in update mode the keywords DATASUM and CHECKSUM are created / updated. This is the very last step before the extension is closed.

The keywords should already exist from the fsd files.

void FitsDalHeader::InsertCommentLine ( const std::string &  comment)

Insert one comment line in the FITS header.

A comment line is a line in the FITS header which has only a comment, no name and no value. It can even be an empty line.

Parameters
[in]commentThe comment to be placed in one line of the FITS header. It can be an empty string to place and empty line in the FITS header. If it is not empty a '/ ' will be placed in front of the text to indicate it as a comment. The maximum length is 68 characters. It will be truncated if it is longer.

void FitsDalHeader::VerifyChecksum ( )
protected

Validates the DATASUM and the CHECKSUM keywords.

This method should be called each time a HDU is opened. It verifies the DATASUM and the CHECKSUM keywords. If one of these keywords are not available a warning is issued via the logger module. If one of these keywords are not correct a runtime_error exception is thrown.

Exceptions
runtime_errorif one or both of the keywords DATASUM and CHECKSUM are not valid.

◆ WriteCurrentStatus()

void FitsDalHeader::WriteCurrentStatus ( const std::string &  filename)

Writes the current status into a new FITS file.

This method should be used only for debugging purpose. Only the FITS table or the FITS image can be copied into the new FITS file. The status of the header keywords is not defined.
A FITS table in the saved FITS file may have more rows as already created in this source table. The values in the additional rows are random.
A FITS image will be copied into the primary array of the new FITS file.

Parameters
[in]filenameof the file with the copied data. The file must not exist before calling this method.
Exceptions
runtime_errorif the new file cannot be created or when the write operation fails.
