Detailed Description

Class used for reading in the log level of a log entry.

Author
Reiner Rohlfs UGE
Anja Bekkelien UGE

It overloads the << operator to take a log level as input. The operator returns an object used to input the rest of the log message. What type of object that is, depends on the log level.

Configures the logger.

The path to the log file can be provided in two ways. 1) By defining a directory. A new log file is created every time the program is run, with a unique file name containing the current time in UTC. 2) By defining the full path to a log file. If this file already exists, logs are appended to it.

Parameters
[in]stdoutLevel(optional) the log level for stdout. Must be "DEBUG", "INFO", "PROGRESS", "WARNING", "ERROR". Set to INFO by default.
[in]stderrLevel(optional) the log level for stderr. Must be "DEBUG", "INFO", "PROGRESS", "WARNING", "ERROR". Set to PROGRESS by default.
[in]alertEmail(optional) the email to which alerts are sent. If this parameter is not provided, no emails are sent
[in]logDirectory(optional) the directory in which a log file is created. If parameter logFile is provided, logDirectory is ignored
[in]logFile(optional) the path to the log file, including directories and file name. If this parameter is provided, parameter logDirectory is ignored
[in]revisionNumber(optional) The archive revision number
[in]processingNumber(optional) The processing number
[in]visitId(optional) the visit id as a string formatted as PRppnnnn-TGoooonn. If this argument is provided, the visit id is included in the file name.
[in]passId(optional) the pass id as a string formatted as PSyymmddhh. If this argument is provided, the pass id is included in the file name.
[in]obsid(optional) the obsid
[in]processingChain(optional) the processing chain. If provided, it is included in the log file name.
Returns
the path to the log file, or an empty string if no log file was created
