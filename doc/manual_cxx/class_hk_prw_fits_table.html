Detailed Description

Class to fill one PRW_HK FITS tables with data of one TM packet or some parameters of one TM packet.

Author
Reiner Rohlfs UGE

The only way to create such a class is to call HkTm2PrwProcessing::getHkPrwFitsTable() for a specific HK TM packet.

void HkPrwFitsTable::addTmPacket ( const OBT &  obt,
const int8_t *  tmPacketData,
uint16_t  maxLength = UINT16_MAX

Writes the data of one TM packet to the next row in the associated FITS tables.

All FITS tables are updated if data of one TM packet shall be written in more than one FITS table.

Parameters
[in]obtThe OBT of the data, as defined in the TM header. The obt has to be greater then the obt of the previous call of this method.
[in]tmPacketDataThe data of a HK TM packet. The index of a parameter in this tmPacketData is defined by the tmOffset attribute of that column in the SCI_PRW_HK* fsd file. The length of the buffer has to be long enough to hold the complete data of one TM packet. The length depends on the TmPacket. There is no check inside if the length is too short!
[in]maxLengthThe maximum length of the parameter in bytes.
Exceptions
runtime_errorif the obt is equal of less than the obt of the previous call.
