Detailed Description

Bad Pixel Map of a Full-Array.

Author
Reiner Rohlfs UGE

The Bad Pixel Map is derived from the dark MandC observations. First the MCO_REP_BadPixelMapFullArray is created. After an inspection by PSO / Instrument Team it will be approved and copied to REF_APP_BadPixelMap. This approved data structure will be provided to the programs of the CHEOPS data processing. The ground calibration also provides an approved Pad Pixel Map Pixels can have following values: -2: totally dead pixel, -1 = partially dead pixel, 0 = good pixel, 1 = hot pixel, 2 = saturated pixel 3 = telegraphic pixel.

◆ getData()

def getData ( )

Returns a numpy array containing the image data.

The image data has the slowest changing axis first and the fastest changing axis last. For example, for an image cube, the first axis corresponds to the index of a 2D image in the cube, and the second and third axis corresponds to the Y and the X axis respectively of the 2D image.

The returned numpy array provides direct access to the memory holding the image data and is not a copy. As long as any numpy arrays referencing this memory exist, the fits image object to which the image data belongs is kept alive.

◆ writeCurrentStatus()

def writeCurrentStatus ( filename  )

Writes the current status into a new FITS file.

This method should be used only for debugging purpose. Only the FITS table or the FITS image can be copied into the new FITS file. The status of the header keywords is not defined.

A FITS table in the saved FITS file may have more rows as already created in this source table. The values in the additional rows are random.

A FITS image will be copied into the primary array of the new FITS file.

Parameters
filenameof the file with the copied data. The file must not exist before calling this method.
Exceptions
runtime_errorif the new file cannot be created or when the write operation fails.
